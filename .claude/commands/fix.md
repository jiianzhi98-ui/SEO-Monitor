# 修复模式 — 按优先级逐项修复

根据上一份 Audit Report 的发现，进入修复模式。

## 修复规则（必须遵守）

1. **一次只修一个问题** — 从 Critical 开始，按严重等级从高到低处理
2. **不要顺便优化其他地方** — 只改与当前问题直接相关的代码
3. **不要引入新的依赖或抽象** — 用最小改动解决问题
4. **修改后必须说明验证方法** — 如何确认修复生效

## 每次修复的输出格式

```
## 修复 #N — [问题简述]

**严重等级：** Critical / High / Medium / Low
**来源：** [Audit Report 中哪位专家发现的]

### 修改内容
- 文件：`path/to/file.tsx`，第 X 行
- 改动说明：[具体改了什么]

### 原因
[为什么这样改能解决问题]

### 影响范围
- 影响了哪些功能：[列出]
- 是否影响其他组件：[是/否，说明]

### 验证方法
1. [具体操作步骤]
2. [预期看到的结果]

### 待修复队列（剩余）
- [ ] Critical: [下一个问题]
- [ ] High: ...
```

## 修复完成后

所有 Critical 修复完成后，输出：

```
## 修复完成总结

已修复：X 个 Critical，X 个 High
待处理：X 个 Medium（可下次处理），X 个 Low（可选）

建议：运行 /audit 再次确认，然后运行 /release 准备上线
```
